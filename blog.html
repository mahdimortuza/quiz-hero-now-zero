1. What is the difference among map, filter, forEach and find?

Answer: In JavaScript, map(), filter(), forEach(), and find() are all array methods used to manipulate arrays. Here are the differences among them:

  • map() - creates a new array with the results of calling a provided function on every element in the original array. It returns a new array with the same length as the original array.
  • filter() - creates a new array with all the elements that pass the test provided by a callback function. It returns a new array that may have fewer elements than the original array.
  • forEach() - calls a provided function once for each element in the array, but it doesn't create a new array. It simply executes a function on each item in the array.
  • find() - returns the value of the first element in the array that satisfies the provided testing function. If no element satisfies the function, it returns undefined.

2. What is the difference among local storage, session storage and cookies?

Answer:

  • Local storage is used save data on the browser. This data is not accessible from another browser. Local storage is more suitable for storing larger amounts of data that only need to be accessed by the client.
  • Session storage is used to store data of a ongoing session. If the user close a session then all of the data are gone.
  • Cookies are more suitable for storing small amounts of data that need to be accessed by both the client and server.

3. What is event loop and how does it work?

Answer: The event loop is a crucial part of JavaScript's runtime that handles the execution of asynchronous code. It's responsible for managing the execution of code in a way that maximizes efficiency and responsiveness.
